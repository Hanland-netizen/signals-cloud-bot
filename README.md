Binance Futures Signals Bot

Автоматизированный скрипт для анализа рынка Binance USDT-M Futures и генерации торговых сигналов на основе импульсных движений, EMA200, RSI и контекстного анализа BTC. Поддерживает многопользовательскую подписку через Telegram и 24/7 работу на облачных платформах, в том числе Railway.

Описание

Приложение выполняет постоянное сканирование USDT-M PERPETUAL инструментов Binance, фильтруя монеты по ликвидности, находя импульсные свечи, анализируя рыночный контекст, рассчитывая уровни входа, стоп-лосса и тейк-профита. Сигналы отправляются всем пользователям Telegram, которые подписались через команду /start.

Бот предназначен для непрерывной работы в облаке, устойчив к ошибкам API за счёт системы повторных запросов и полностью автономен.

Основные возможности

Сканирование всех USDT-M PERPETUAL инструментов Binance.

Фильтрация активов по минимальному торговому объёму (24h quote volume).

Обнаружение импульсных свечей с использованием параметров тела и объёма.

Расчёт EMA200 и RSI(14) для определения трендов и подтверждающих условий.

Фильтр по контексту BTC (цена, EMA200, RSI, 24h изменение).

Определение направлений входа, стоп-уровней, тейк-профита и рекомендуемого плеча.

Ограничение количества сигналов: максимальное число в день и за один проход.

Поддержка Telegram команд: /start, /stop, /status, /help.

Управление подписчиками через файл subscribers.txt.

Автоматическая рассылка сигналов всем пользователям.

Поддержка развёртывания в облаке (Railway) без необходимости локального запуска.

Установка и локальный запуск
1. Клонирование репозитория
git clone https://github.com/Hanland-netizen/signals-cloud-bot
cd signals-cloud-bot

2. Создание виртуального окружения
python3 -m venv venv
source venv/bin/activate     # Windows: venv\Scripts\activate

3. Установка зависимостей
pip install -r requirements.txt

4. Настройка переменных окружения

Перед запуском необходимо определить:

export TG_BOT_TOKEN="ваш_токен_бота"
export TG_CHAT_ID="ваш_чат_id"     # необязательно

5. Запуск
python binance_bot.py

Развёртывание на Railway
1. Подготовка репозитория

Убедитесь, что в проекте присутствуют:

binance_bot.py

requirements.txt

Файл автоматически определит интерпретатор Python.

2. Создание проекта на Railway

Авторизоваться через GitHub.

Выбрать "New Project" → "Deploy from GitHub".

Подключить репозиторий signals-cloud-bot.

3. Настройка переменных окружения

Во вкладке Variables добавить:

TG_BOT_TOKEN = <токен_вашего_telegram_бота>
TG_CHAT_ID   = <необязательный id администратора>

4. Команда запуска

Во вкладке Deploy / Settings задать:

python binance_bot.py


Railway автоматически установит зависимости и запустит процесс.
Сервис будет сам перезапускаться при сбоях.

Конфигурация

В начале файла binance_bot.py присутствует блок CONFIG, содержащий параметры:

MIN_QUOTE_VOLUME — минимальный торговый объём.

RISK_REWARD — коэффициент RR.

MIN_RISK_PCT — минимальный размер стопа (%).

MAX_SIGNALS_PER_DAY — дневной лимит сигналов.

MAX_SIGNALS_PER_SCAN — лимит сигналов за одно сканирование.

BTC_FILTER_ENABLED — включение фильтра по BTC.

Изменение этих значений позволяет адаптировать логику под торговые предпочтения.

/stop — отмена подписки.

/status — текущее состояние бота.

/help — описание команд и логики.
